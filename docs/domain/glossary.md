# Domain Glossary

| Term | Definition | Bounded Context | Example Usage |
| --- | --- | --- | --- |
| Workspace | A named, self-contained organization environment that groups all users, channels, and messages. The top-level tenant boundary. | Identity & Access | "The engineering team created a Workspace called 'Acme Corp' to centralise all internal communication." |
| WorkspaceMember | A user who has been granted access to a Workspace and assigned a role (owner, admin, or member). | Identity & Access | "After accepting the invite link, Sarah became a WorkspaceMember with the 'member' role." |
| WorkspaceRole | The permission level assigned to a WorkspaceMember — one of `owner`, `admin`, or `member`. | Identity & Access | "Admins can archive channels; only the owner can delete the Workspace itself." |
| WorkspaceInvite | A time-limited, token-based link that allows a new user to join a Workspace as a member. | Identity & Access | "The admin generated a WorkspaceInvite that expires in 7 days and shared it with new hires." |
| Channel | A named, topic-scoped conversation space inside a Workspace. Can be public (visible to all members) or private (invite-only). | Messaging | "The #announcements Channel is public and read-only for non-admins; #backend is a private Channel for the API team." |
| Message | An immutable unit of content posted by a WorkspaceMember to a Channel or Thread. The core domain object. | Messaging | "Alice posted a Message in #general asking about the deployment schedule." |
| Thread | A sub-conversation anchored to a parent Message. Keeps replies contextually grouped without cluttering the main Channel. | Messaging | "Bob replied in the Thread on Alice's deployment question rather than posting to #general." |
| DirectMessage (DM) | A private, one-to-one (or small-group) conversation between WorkspaceMembers that exists outside any Channel. | Messaging | "Carol sent a DirectMessage to Dave to share confidential details before the team meeting." |
| Reaction | An emoji acknowledgement a WorkspaceMember attaches to a Message to express sentiment without posting a reply. | Messaging | "Eve added a 👍 Reaction to the release announcement to confirm she had read it." |
| Presence | The real-time online/away/offline status of a WorkspaceMember, broadcast to all members of their Workspace. | Presence | "Frank's Presence indicator turned grey when he closed the browser tab, signalling he was offline." |
| Notification | An in-app or push alert delivered to a WorkspaceMember when they are @mentioned or a keyword they follow appears in a Message. | Notification | "Grace received a Notification when @channel was used in #ops, even though she had the tab closed." |
| Mention | The act of referencing a specific WorkspaceMember (`@username`), group (`@channel`, `@here`), or Channel (`#channel-name`) inside a Message, triggering Notifications. | Notification | "Henry used an @here Mention to alert only currently-active members about the incident." |
| ChannelMembership | The record that a WorkspaceMember has joined a specific Channel. Required for private Channels; implicit for public ones. | Messaging | "Irene was added to ChannelMembership for #design-review after requesting access from the admin." |
| MessageHistory | The ordered, append-only log of Messages in a Channel or Thread, used for rendering the conversation view. | Messaging | "The client fetches the last 50 items of MessageHistory on initial load, then receives new Messages via WebSocket." |
| RefreshToken | A long-lived credential stored server-side (hashed) that allows a user to obtain a new access token without re-authenticating. | Identity & Access | "When the access token expired, the client silently exchanged its RefreshToken for a new one." |
| AccessToken | A short-lived JWT proving the identity of an authenticated WorkspaceMember, sent on every API request. | Identity & Access | "The middleware validates the AccessToken on each request and rejects it if it has expired." |
| Slug | A URL-safe, human-readable identifier for a Workspace (e.g., `acme-corp`), used in routing instead of a numeric ID. | Identity & Access | "The Workspace 'Acme Corp' is accessible at `/w/acme-corp` using its Slug." |
| ChannelArchive | The act of making a Channel read-only and hidden from the default sidebar, preserving history without allowing new posts. | Messaging | "The admin triggered a ChannelArchive on #2024-project after the engagement ended." |
| WebSocketSession | An active, persistent connection between a client and the server used to push real-time events (new Messages, Presence updates) without polling. | Real-Time | "Each browser tab maintains one WebSocketSession; reconnection is automatic on network drop." |
| EventEnvelope | The wrapper structure around every real-time event sent over a WebSocketSession, carrying a type, payload, and sequence number. | Real-Time | "The client uses the sequence number in the EventEnvelope to detect and re-request any missed Messages after a reconnect." |
